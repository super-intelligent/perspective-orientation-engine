## THREE ENHANCEMENT LAYERS

### 1. SEMANTIC COMPRESSION LABELS (Claim Archetypes)
**File:** `FEATURE-SEMANTIC-COMPRESSION.md`
**Priority:** HIGH — foundational to all other enhancements
**Classification:** 🟡 YELLOW minimum (touches output format + prompt)

Adds a short, intuitive human-readable label to each claim that compresses
the tensor meaning into a single orientation descriptor.
Example: Instead of just "UL · Complex · Abductive · High" the user also sees
"Identity Destabilization" — which binds the tensor into perception.

### 2. TENSOR TENSION SIGNALING (Multi-Quadrant Placement)
**File:** `FEATURE-TENSION-SIGNALING.md`
**Priority:** HIGH — Russell directed this begin immediately, not deferred
**Classification:** 🔴 RED (touches extraction prompt + schema + visualization)

Shows primary placement PLUS visible tension toward other axes.
Not multi-classification. Not false precision. Tension visibility.
Example: Primary LL, Tension toward LR and UL.

### 3. GRAVITATIONAL CENTER EXPLANATION
**File:** `FEATURE-GRAVITATIONAL-STRUCTURE.md`
**Priority:** HIGH — Russell identified this as "the most exciting part"
**Classification:** 🔴 RED (touches extraction prompt + new output structure)

Reveals WHY a claim became the gravitational center. Shows which claims
feed into it, by what gravity type, and the recursive feedback loops
that make it structurally organizing.

## DESIGN PRINCIPLES (from the analysis)

1. These are NOT features bolted on. They are layers that make the tensor
   perceptually intelligible.
2. Semantic labels are ORIENTATION DESCRIPTORS, not interpretations.
3. Tension signals preserve HONESTY of placement without false precision.
4. Gravitational structure shows HOW meaning organizes itself, not WHY
   it should.
5. TRUE-BY-ASSUMPTION is never violated. No truth evaluation. No prescription.
6. The system teaches users to SEE — that is the function of these layers.
7. Primary placement gives CLARITY. Secondary tendencies give HONESTY.
   Together they reveal why certain claims become gravitational centers.

## IMPLEMENTATION ORDER (Recommended)

1. Semantic Compression Labels — smallest prompt change, highest impact
2. Tension Signaling — requires schema extension + prompt change
3. Gravitational Structure — requires new output section + relationship logic

All three can be built incrementally on the existing extraction pipeline.
