## WHAT THIS IS

A short human-readable label added to each extracted claim that compresses
the tensor meaning into a single intuitive orientation descriptor.

This is NOT classification. NOT judgment. NOT interpretation.
It is: "What kind of orientation event is this, experientially?"

## WHY THIS MATTERS

The raw tensor output tells the user WHERE claims are:
  UL · Complex · Abductive · High

The semantic compression label tells the user WHAT KIND of thing it is:
  "Identity Destabilization"

The label does not replace the tensor. It BINDS it into perception.
It acts as a cognitive handle on multi-dimensional placement.

## DOCTRINE COMPLIANCE

These labels must be:
- Orientation descriptors, NOT interpretations
- Structurally derived from the tensor placement, NOT from content judgment
- Consistent with TRUE-BY-ASSUMPTION (no truth evaluation embedded)
- Consistent with vocabulary governance (no "analysis" language)

## WHAT THE AI MUST PRODUCE (per claim)

Current output per claim:

    "id": "c1",
    "text": "...",
    "quadrant": "UL",
    "domain": "Complex",
    "reasoning": "Abductive",

## LABEL CHARACTERISTICS (Design Constraints)

Good archetype labels share these properties:
1. Two to four words maximum
2. Structurally descriptive, not emotionally loaded
3. Could apply to ANY brain dump with similar tensor placement
4. Do not embed truth claims or moral positioning
5. Compress quadrant + domain + reasoning into a single intuitive phrase

## WHAT CHANGES IN THE SYSTEM

### Extraction Prompt
Add instruction to the system prompt requiring Claude to generate an
"archetype" field per claim. The instruction must emphasize:
- Orientation descriptor, not interpretation
- Structurally derived from tensor placement
- Two to four words
- No truth evaluation or prescription embedded

### Output Schema (Zod)
Add `archetype: z.string()` to the claim schema.

### UI (OrientResults.tsx)
Display the archetype label prominently on each claim card — likely as
the card heading or subheading, with the tensor badges below it.

### Display Priority
The archetype label should be the FIRST thing the user reads on each card.
The tensor badges provide supporting detail underneath.
This inverts the current display hierarchy (currently badges-first).

## INTERACTION WITH OTHER FEATURES

- Tension Signaling: archetype labels help explain WHY a claim has tension
  toward other quadrants
- Gravitational Structure: archetype labels make the gravity relationship
  descriptions more intuitive (e.g., "Constraint Lock feeds Systemic
  Fracture Realization" reads better than "C10 feeds C14")

## Q1 ANSWER: THE GENERATIVE PRINCIPLE (Critical — This Is the Prompt Rule)

A good archetype label is NOT a content summary.
It is: a compression of WHAT KIND OF ORIENTATION EVENT is happening.

Not content → FUNCTION of the claim in the field.

### The Hidden Mechanism (3-Part Fusion)

Each label is derived from fusing:
1. Quadrant = WHERE it lives
2. Domain = HOW stable/knowable it is
3. Reasoning = HOW the observer is constructing it

Then translated into: "What is this claim DOING in the orientation field?"

### Example (C1)
- Upper-left → interior experience
- Complex → unstable, emergent
- Abductive → trying to explain something unclear

This becomes NOT "Teacher feels lost"
But: "Identity Destabilization"

Because the FUNCTION is: internal self-structure losing coherence
under uncertainty.

### THE PROMPT INSTRUCTION (ChatGPT's exact recommendation)

```
Generate a 2-4 word archetype label that describes the FUNCTION of
the claim in the orientation field, not its content.

The label should answer:
"What kind of orientation event is this?"

Derive it from:
- quadrant (where it lives)
- domain (stability of cause-effect)
- reasoning mode (how the claim is constructed)

Avoid:
- restating the claim
- emotional language
- moral implication

Prefer:
- structural, process-oriented phrasing
- verbs or nouns that imply system behavior
```

### Key Distinction: These Are NOT Categories — They Are Field Roles

Like chess pieces:
- Not "this is a knight"
- But "this is a fork threat"

### Confidence Assessment
- STRONG (lock this): labels must be functional compressions
- FLEXIBLE: exact wording vocabulary can vary by context

## Q2 ANSWER: ARCHETYPE PATTERN LANGUAGE PER QUADRANT

There IS a pattern language per quadrant. Not rigid taxonomy, but
recurring "shapes of events."

### Upper Left (Interior / Individual)
Theme: STABILITY OF SELF-MODEL
Common archetype shapes:
- Destabilization (Identity Destabilization)
- Ambiguity (Value Ambiguity Recognition)
- Avoidance (Avoidance Threshold Breach)
- Realization (Systemic Fracture Realization)
→ UL = state transitions inside the observer

### Upper Right (Exterior / Individual)
Theme: OBSERVABLE BEHAVIOR AND SIGNAL DETECTION
Common shapes:
- Detection (Pattern Anomaly Detection)
- Observation (Rule Violation Observation)
- Execution patterns (System Gaming)
→ UR = what can be pointed at

### Lower Left (Interior / Collective)
Theme: MEANING, NORMS, INTERPRETATION
Common shapes:
- Drift (Norm Drift Assertion)
- Conflict (Interpretive Conflict Injection)
- Ambiguity (Authority Ambiguity Signal)
→ LL = shared meaning destabilizing or shifting

### Lower Right (Exterior / Collective)
Theme: SYSTEMS, CONSTRAINTS, OUTPUTS
Common shapes:
- Breakdown (Institutional Vacuum)
- Constraint (Constraint Lock)
- Redistribution (Responsibility Displacement)
- Outcome signals (Inequity Signal)
→ LR = system behavior under pressure

### KEY INSIGHT FROM CHATGPT:
"You are not labeling CONTENT TYPES.
You are labeling FAILURE MODES AND TRANSITIONS in each quadrant."

### Confidence Assessment
- STRONG: each quadrant has recurring "event shapes"
- FLEXIBLE: labels are still context-generated, not pre-selected

The vocabulary will shift across different brain dumps.

You will always see:

# CHATGPT REFINEMENT PASS 2 — Integration (2026-03-21)

## R1: ARCHETYPE LABEL CALIBRATION

### Critical Expansion: Not All Archetypes Are Failures

Four structural types of archetype labels:
1. DESTABILIZATION → something breaking
2. CONSTRAINT → something limiting
3. TRANSITION → something shifting
4. STABILIZATION / PATTERNING → something holding or organizing

The system needs ALL FOUR or it becomes biased toward breakdown narratives.
This is a prompt instruction requirement.

### Calibration Example 1 — Startup Founder

Claim: "Every strategic decision turns into a debate about control."
→ LL · Complex · Abductive · High

GOOD: "Control Conflict Recurrence"
- "Conflict" → LL (shared meaning tension)
- "Recurrence" → Complex (pattern over time)
- "Control" → content compressed into structural theme
- Describes a repeating relational dynamic

BAD: "My Co-Founder Is Difficult"
- Content-based, not structural
- Emotionally framed, not transferable
- No tensor compression

### Calibration Example 2 — Medical Diagnosis

Claim: "The test results are inconclusive, but my doctor still wants
to start treatment."
→ LR · Complicated · Abductive · High

GOOD: "Premature Intervention Signal"
- LR → system/medical protocol
- Complicated → requires expertise
- Abductive → acting under uncertainty
- "Signal" → observational, not judgment

BAD: "Doctor Is Wrong"
- Truth claim, violates doctrine
- Collapses orientation into evaluation

### Calibration Example 3 — Project Manager

Claim: "We keep changing priorities faster than teams can execute."
→ LR · Chaotic · Inductive · High

GOOD: "Priority Volatility Pattern"
- LR → system behavior
- Chaotic → instability
- Inductive → pattern observed over time
- "Volatility" captures instability without judgment

BAD: "Leadership Is Messy"
- Vague, interpretive, not structurally descriptive
- Does not map to tensor

### Calibration Example 4 — Personal / Interior

Claim: "I thought I wanted this job, but now I feel disconnected
from it."
→ UL · Complex · Inductive · High

GOOD: "Value Alignment Drift"
- UL → interior state
- Complex → unfolding realization
- Inductive → pattern from experience
- "Drift" captures gradual change

BAD: "I Hate My Job"
- Emotional shorthand, no structural insight, not transferable

### THE CALIBRATION RULE

A GOOD label always:
- Removes WHO/WHAT
- Preserves WHAT IS HAPPENING STRUCTURALLY
